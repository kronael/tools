# Short-form Video Scripts

## Voice

- Spoken aloud — read every line out loud before shipping; trip on a word, rewrite it.
- Concrete demo, not pitch. "Look at this" beats "let me tell you about".
- One viewer ear, not a crowd. "You" never "everyone".
- No emojis in the script (they're for captions, set by editor). No bracketed stage directions in the read-aloud lines.

## Structure (45-75 seconds spoken, ~110-180 words)

1. **Hook (0-3s)**: ONE concrete contrast or claim. Not a question. Not a setup.
2. **Demo beat (3-25s)**: show the thing working. Each line = one visible action.
3. **Why it matters (25-45s)**: the pain it removes, in user vocabulary.
4. **Payoff (45-60s)**: the moment the viewer "gets it".
5. **CTA (last 3-5s)**: ONE action. Link, follow, or try.

Pacing target: ~2.5 words per second spoken. 60 seconds ≈ 150 words.

## Rules

- ALWAYS open with action or contrast in the first line — NEVER "Hey guys", "in this video", "today I'll show".
- ALWAYS write for the ear: short sentences, monosyllables, no nested clauses.
- ALWAYS put the strongest demo beat in the first 10 seconds — drop-off is brutal there.
- ALWAYS end on one CTA, never two.
- NEVER use "powerful", "seamless", "incredible", "amazing", "revolutionary", "game-changer".
- NEVER explain what the tool/product is before showing it — show first, name after.
- NEVER read out a URL aloud — put it in caption/description only.
- NEVER write filler ("so basically", "the thing is", "you know") — cut on the second pass.

## Format on the page

```
[B-roll: terminal closeup, tool launching]
The build that took six minutes now takes nine seconds.

[Cut: side-by-side, old vs new]
Same project, same machine. No caching tricks.

[Voice over screen capture]
The trick is — it doesn't compile what didn't change.
...
```

- Bracketed `[direction]` lines describe what's on screen. NOT spoken.
- Plain lines are spoken verbatim. Read each aloud.
- Empty line between beats.

## Length variants

| platform | target | hard cap |
|---|---|---|
| X/Twitter video | 30-45s | 140s |
| YouTube Shorts | 45-60s | 60s |
| TikTok | 30-60s | 60s (algorithm) / 180s (hard) |
| Reels | 30-60s | 90s |

If forced longer, split into a 2-video series with a hook bridge ("part one was the build; part two is the deploy").

## Anti-patterns to refuse

- "Three things you need to know about X" — listicle bait, conversion is poor.
- "Did you know that…" — opens with a question; viewer thumbs past.
- "Like and subscribe" before the payoff — kills retention; CTA goes LAST.
- Walking through a feature menu — show outcomes, not menus.
